WebAR Hologram Experience (2026 Edition)

Інтерактивна WebAR-голограма людини в повний зріст (1.5м), реалізована за допомогою **MindAR** та **A-Frame**. 
 Швидкий старт

1. Відкрийте [Посилання на ваш GitHub Pages] на мобільному пристрої.
2. Надайте доступ до камери (обробка відбувається локально на пристрої).
3. Наведіть камеру на **Маркер А5** (див. нижче).
4. Після появи голограми натисніть **"🔊 Увімкнути звук"**.
5. Ви можете вільно ходити навколо голограми — вона зафіксована у просторі.

Технологічний стек

* **MindAR 1.2.x:** Стабільний трекінг зображень.
* **A-Frame 1.4+:** VR/AR фреймворк для WebGL.
* **Alpha Mask Shader:** Кастомний шейдер для відтворення прозорості через Side-by-Side (RGB + Alpha) відео.
* **Three.js:** Математика для "прив'язки" об'єкта до світових координат після детекції.
 Структура репозиторію

- `/assets/targets.mind` — скомпіровані точки детекції для AR.
- `/assets/hologram_alpha.mp4` — відео-контент (ліва частина — колір, права — маска).
- `/assets/marker_preview.jpg` — файл маркера для друку.
- `index.html` — ядро програми (сцена, шейдери та логіка фіксації).

 Маркер для активації

Для найкращого досвіду використовуйте маркер формату **А5**. Ви можете завантажити його тут:
 [Завантажити маркер](./assets/marker_preview.jpg)

> **Порада 2026:** Покладіть маркер на підлогу з вираженою текстурою (дерево, бетон, килим). Це допоможе камері телефону краще утримувати 1.5-метрову фігуру стабільною.

 Особливості реалізації

1.  **Fixed World Position:** Після знаходження маркера голограма "відв'язується" від нього. Ви можете прибрати маркер або ходити навколо об'єкта.
2.  **Billboard Effect:** Завдяки компоненту `look-at`, 2D-відео завжди повернуте до користувача, створюючи ілюзію об'єму.
3.  **High-Fidelity Alpha:** Використання маски замість Chroma Key дозволяє зберегти деталі волосся та позбутися "зелених ореолів".
4.  **Privacy First:** Додаток не передає дані з камери на сервери. Все працює через WebAssembly у вашому браузері.


 Як розгорнути цей проєкт

1. Форкніть цей репозиторій.
2. Завантажте свої файли в папку `/assets`.
3. Перейдіть у **Settings > Pages** вашого репозиторію.
4. Оберіть гілку `main` та папку `/root`.
5. Зачекайте 1-2 хвилини, поки GitHub згенерує HTTPS-посилання.
